# Consultations

A **consultation** is a planned care or follow-up event.  
It involves:

- a **practitioner** (the person in charge)
- a **patient** (the player)
- a possible link to a **pathology** (injury/illness)

## Create a consultation

### 1) Select the player (patient)
The appointment is linked to the relevant player.

### 2) Assign the practitioner
You select the person who will carry out the consultation.

### 3) Choose a consultation type
Types are configurable according to your organization.

### 4) Set confidentiality
You choose who can see the consultation details (see below).

### 5) (Optional) Link it to a pathology
This is recommended to keep a clear history between the health incident and the actions taken.

## Confidentiality: visibility levels

The consultation can be visible to:

1. **Doctors only**
2. **Doctors + paramedical staff**
3. **Doctors + paramedical staff + restricted access (sports staff)**
4. **Private** (creator + practitioner)
5. **Specific** (named list)

### Masking rule (“Busy”)

If a user does not have the rights, they will only see a “BUSY” time slot, with no details.

## Link a consultation to a pathology (3 cases)

- **Link an existing pathology**: select from the player’s open (or recent) pathologies.
- **Create the pathology from the consultation**: immediate creation, then automatic link.
- **Link it later**: the link can also be added after the consultation.
