# Medical notes

## What is it for?
**Medical notes** make it possible to keep clear, structured records of information linked to medical follow-up, **directly attached** to:

- a **consultation**
- a **pathology**

✅ A medical note is always **linked** to a consultation or a pathology.  
❌ A note with “no link” does not fit this model.

---

## What you can do

- **Create** a note linked to a consultation or a pathology.
- **Find** all notes for a consultation / pathology.
- **Manage visibility** (sharing) while respecting security rules.

---

## Who can create / edit a note?

### If the note is linked to a consultation
✅ **Only the assigned doctor** for the consultation can **create and edit** the note.

### If the note is linked to a pathology
✅ Anyone with **access to the pathology** can **create and edit** the note.

:::note
You do not manually choose “all doctors” vs “team doctors”.  
👉 It is **automatically inferred** from the linked entity (consultation / pathology).
:::

---

## How do I find the notes for a consultation or a pathology?

To display the notes for an entity, you need to run a **notes search** using:

- `related.type` = **Consultation** or **Pathology**
- `related.id` = the entity identifier

**Examples:**

- Consultation notes → `Consultation + consultationId`
- Pathology notes → `Pathology + pathologyId`

---

## How do I create a linked note? (system rules)

When creating a note, these rules apply:

- **`related` is required** (consultation or pathology)
- **`type` = `ams`**
- **`patient` always** if available (if the patient is known)
- **`moduleId` is required**
