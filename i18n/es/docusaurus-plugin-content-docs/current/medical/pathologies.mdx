# Patologías

Una **patología** es la ficha central del módulo médico.  
Representa un incidente de salud que afecta a un jugador:

- **Lesión**: daño físico.
- **Enfermedad**: afección orgánica o viral.

## Creación de una patología

### Seleccionar al jugador
Vinculas la patología a un jugador.

### Elegir el tipo: lesión o enfermedad
El tipo determina la estructura de la clasificación.

### Introducir el diagnóstico mediante una clasificación estandarizada
La patología se estructura según una clasificación médica (OSIICS):

- Lesión: Región → Tipo → Diagnóstico.
- Enfermedad: Sistema → Etiología → Diagnóstico.

### Referencia de la clasificación
- Historial de versiones de la clasificación OSIICS: https://www.johnorchard.com/osiics-version-history.html

### Introducir las fechas clave
- Fecha de declaración (aparición).
- Fecha estimada de retorno (previsto deportivo).
- Fecha real de retorno (retorno efectivo).

### Completar la información útil para el seguimiento
Según el tipo, puedes documentar el contexto y los elementos necesarios para la atención del caso, por ejemplo una recaída o el mecanismo.

## Reglas de disponibilidad

La creación de una patología cambia automáticamente el estado del jugador a “Lesionado” (indisponible).

La patología se considera **cerrada** cuando se rellena la **fecha real de retorno**.

El módulo gestiona la indisponibilidad deportiva, no una baja laboral administrativa.

## Notas y comunicación

Para proteger la confidencialidad, las notas se separan en 4 zonas:

- **Nota médica**: solo médicos.
- **Nota paramédica**: médicos + paramédicos.
- **Nota global**: visible para los perfiles autorizados, incluido acceso restringido.
- **Nota jugador**: visible para el jugador si la compartición está activada.

## Compartir con el jugador

La compartición hacia la aplicación del jugador es **activable** y **granular**:

- hacer visible la patología: sí / no.
- hacer visible la nota jugador: sí / no.
