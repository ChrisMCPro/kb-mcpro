# Notas médicas

## ¿Para qué sirve?
Las **notas médicas** permiten conservar un registro claro y estructurado de la información relacionada con un seguimiento médico, **siempre vinculada** a una:

- **consulta**
- **patología**

✅ Una nota médica siempre está **asociada** a una consulta o a una patología.  
❌ Una nota “sin vínculo” no entra en este funcionamiento.

---

## Qué puedes hacer

- **Crear** una nota vinculada a una consulta o a una patología.
- **Encontrar** todas las notas de una consulta / patología.
- **Gestionar la visibilidad** (compartición) respetando las reglas de seguridad.

---

## Quién puede crear o modificar una nota

### Si la nota está vinculada a una consulta
✅ **Solo el médico asignado** a la consulta puede **crear y modificar** la nota.

### Si la nota está vinculada a una patología
✅ Cualquier persona con **acceso a la patología** puede **crear y modificar** la nota.

:::note
No se elige manualmente entre “todos los médicos” y “médicos del equipo”.  
👉 Se **deduce automáticamente** según la entidad vinculada (consulta / patología).
:::

---

## Cómo encontrar las notas de una consulta o una patología

Para mostrar las notas de una entidad, hay que hacer una **búsqueda de notas** a partir del vínculo:

- `related.type` = **Consultation** o **Pathology**
- `related.id` = el identificador de la entidad

**Ejemplos:**
- Notas de consulta → `Consultation + consultationId`
- Notas de patología → `Pathology + pathologyId`

---

## Cómo crear una nota vinculada

Al crearla, la nota debe respetar estas reglas:

- **`related` obligatorio** (consulta o patología).
- **`type` = `ams`**.
- **`patient` siempre** si está disponible.
- **`moduleId` obligatorio**.
