# Consultations

Une **consultation** est un acte de soin ou de suivi **planifié**.  
Elle implique :
- un **praticien** (personne en charge)
- un **patient** (le joueur)
- un lien possible avec une **pathologie** (blessure/maladie)

## Créer une consultation


### 1) Sélectionner le joueur (patient)
Le rendez-vous est rattaché au joueur concerné.

### 2) Désigner le praticien
Vous sélectionnez la personne qui réalise l’acte.

### 3) Choisir un type de consultation
Les types sont configurables selon votre organisation.

### 4) Définir la confidentialité
Vous choisissez qui peut voir le détail de la consultation (voir ci-dessous).

### 5) (Optionnel) Relier à une pathologie
Recommandé pour garder un historique clair entre l’incident de santé et les actes réalisés.


## Confidentialité : niveaux de visibilité

La consultation peut être visible :
1. **Médecins uniquement**
2. **Médecins + paramédicaux**
3. **Médecins + paramédicaux + accès restreint (encadrement sportif)**
4. **Privé** (créateur + praticien)
5. **Spécifique** (liste nominative)

### Règle de masquage (“Occupé”)


Si un utilisateur n’a pas les droits, il voit uniquement un créneau “OCCUPÉ”, sans aucun détail.


## Lier une consultation à une pathologie (3 cas)

- **Lier une pathologie existante** : sélection parmi les pathologies ouvertes (ou récentes) du joueur
- **Créer la pathologie depuis la consultation** : création immédiate, puis lien automatique
- **Lier a posteriori** : lien possible même après la consultation
